# 春季读书会资料包｜数据自检清单

> 本清单根据 2026-07-20 口径确认结果整理。
> 用于每次新建活动资料包时，在发布前逐项自查。不作为修复记录。
>
> 严重程度：
> - **🔴 必须通过** — 不通过则数据不可用，不能交付
> - **🟡 建议通过** — 不通过会降低数据可信度，应尽量满足
> - **⚪ 可选** — 有则更好，无则不影响基本使用

---

## 一、字段完整性

### 1.1 数值字段必须有单位列

- **检查内容**：所有含数字的字段，相邻处必须有"单位"列或表头中注明单位。
- **合格标准**：每个数值单元格均可独立解读，无需依赖备注或上下文推断单位。
- **严重程度**：🔴 必须通过
- **例外情况**：如果该活动所有数值使用同一单位，可以在表头或文件说明中统一声明一次，不必每列重复。

### 1.2 评分字段必须标注维度和分值范围

- **检查内容**：评分类的列名或相邻列中必须说明评分维度和满分值。
- **合格标准**：看到评分数值即可知道"评的是什么、满分是多少"。
- **严重程度**：🟡 建议通过
- **例外情况**：如果该活动只有一个评分项且已在群公告/README 中统一说明，表头可不重复，但仍建议直接在表中标明。

### 1.3 二值字段不能出现第三值

- **检查内容**：明确设计为二值的字段（例如"是否到场""是否完成""已确认/未确认"），取值集合不能出现第三种写法。
- **合格标准**：全表该字段所有单元格的取值只有两种。
- **严重程度**：🔴 必须通过
- **例外情况**：该字段在设计阶段确实允许三态（例如"是/否/待确认"），则必须先在表头或备注中说明允许的取值集合。默认二值字段不允许出现非二值。

### 1.4 必填字段不允许空值

- **检查内容**：关键字段（姓名、金额、数量等）不允许空白单元格。
- **合格标准**：所有必填列无空值。
- **严重程度**：🔴 必须通过
- **例外情况**：备注类字段（例如"备注""补充说明"）可空。如果部分数据暂缺，应在表中填入占位符（如"待补充""未知"）并在备注中说明原因，不允许留空。

## 二、版本管理

### 2.1 同一份产出只能有一个"正式版"

- **检查内容**：同一主题的文档（会议纪要、方案、照片等）在正式目录中只能保留一个版本。
- **合格标准**：目录中不存在文件名冲突的"最终版""真的最终""正式版V2"等多份并列文件。
- **严重程度**：🔴 必须通过
- **例外情况**：历史版本应统一移到"待确认"或"历史版本"子目录，文件名加注版本号或日期区分。根目录和职能目录下只保留当前有效版本。

### 2.2 版本链必须连续

- **检查内容**：如果文件有版本号（v1、v2、v3），编号不能跳号（不应出现 v1→v3 中间缺 v2 的情况）。
- **合格标准**：版本链连续或版本说明中解释了跳号原因。
- **严重程度**：⚪ 可选
- **例外情况**：如果 v2 因合并或废弃从未实际生成，可以在 v3 的版本说明中注明。

### 2.3 废弃文件必须明确标记或移动

- **检查内容**：文件名或目录位置能明确区分"有效"和"废弃/待确认"两种状态。
- **合格标准**：所有废弃或不明确用途的文件存放在统一目录（如"99_待确认"），根目录和职能目录下不夹杂废弃文件。
- **严重程度**：🟡 建议通过
- **例外情况**：如果项目很小（文件总数<10），允许在文件名加前缀"废弃-"或"旧版-"标记，不必单独建目录。

## 三、数字口径

### 3.1 同一指标在不同文件中引用时口径一致

- **检查内容**：方案中的目标人数、物料清单中的预估人数、预算表中的计算基准人数，三者应使用同一口径。
- **合格标准**：所有引用到人数的文件使用同一个基准数字（或明确说明各自基准的差异）。
- **严重程度**：🔴 必须通过
- **例外情况**：如果活动阶段不同（如报名目标 vs 到场预估），应在各文件中注明自己的计算口径，且差异在合理范围内（到场率不低于 60%）。

### 3.2 与人数挂钩的物资数量应有计算公式

- **检查内容**：按人数采购的物资（茶歇、矿泉水、资料卡等）应有单价×人数的计算依据，不能只有一个笼统的总数。
- **合格标准**：表格中能看出"每人配多少"或"单价×数量"。
- **严重程度**：🟡 建议通过
- **例外情况**：一口价项目（如场地费）不需公式，但应在备注中说明该费用是否与人数无关。

### 3.3 预算表应含控制上限和合计行

- **检查内容**：预算表需要有合计行和预算上限标注。
- **合格标准**：预算表在最后一行显示合计金额，并标注预算上限（如"预算上限：8000 元"）。
- **严重程度**：🟡 建议通过
- **例外情况**：临时草稿阶段的预算表允许暂时不含合计行，但进入"已确认"状态前必须补上。状态标注为"待更新"的项目必须有明确的更新截止时间。

## 四、结论引用

### 4.1 复盘/结论与数据表可交叉验证

- **检查内容**：复盘文档中提到的趋势或结论（如"报名转化不错""某渠道效果好"）应能在对应的数据表中找到直接或间接支撑。
- **合格标准**：对于复盘结论，能在引用数据中给出计算依据或直接数据点引用。
- **严重程度**：🔴 必须通过
- **例外情况**：如果复盘结论基于的数据不在当前文件夹中（如另有完整数据集），应在复盘中注明数据来源和范围。

### 4.2 术语在不同文件中保持一致

- **检查内容**：相同概念在不同文件中使用相同的名称（如"资料包"指代同一对象）。
- **合格标准**：跨文件引用同一对象时，名称表述一致，不会产生歧义。
- **严重程度**：🟡 建议通过
- **例外情况**：对外口径和对内口径可不同（如对外称"伴手礼"、对内称"资料包"），但应在各自文件中说明等价关系。

---

## 五、目录结构

### 5.1 按用途分类建目录，不使用自然状态名称

- **检查内容**：目录名称反映内容用途而非文件状态（用"01_活动策划"而非"文档草案"）。
- **合格标准**：任何人都能根据目录名判断该文件夹里放什么。
- **严重程度**：🟡 建议通过
- **例外情况**：可保留一个"99_待确认"目录存放状态不确定的文件。

### 5.2 每个目录下的根级应有 README 或索引

- **检查内容**：大于 5 个文件的文件夹应有 README 或索引文件说明结构。
- **合格标准**：README 列出子目录用途和文件概览，或索引文件列出所有文件及备注。
- **严重程度**：⚪ 可选
- **例外情况**：文件少于 5 个且命名自解释的目录可不加。
