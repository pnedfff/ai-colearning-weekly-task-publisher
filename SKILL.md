# AI共学群每周任务发布器

## 目标

把 AI 共学群的每周主题稳定发布成一套可执行内容：群公告、任务说明、操作步骤、可复制提示词、打卡提交模板、完成标准和复盘问题。

这个 skill 负责“发布方法”，每周具体任务放在 `references/weeks/` 里。后续更新周任务时，不要新建一个单周 skill；只新增或更新对应周次文件。

固定发布仓库是 `pnedfff/ai-colearning-weekly-task-publisher`。除非用户明确更改目标，否则不要把周任务、评论区或统计写入学员的 Fork 或其他同名仓库。

## 两个固定指令

### 发布本周作业

当用户发送本周素材并说“发布本周作业”时，直接完成以下流程，不要求用户再提供周次或 GitHub 操作步骤：

1. 读取当前日期和 `references/calendars/<半年度>.md`，确定当前周次与日期范围。日期能唯一匹配时不要追问周次。
2. 使用用户本轮或紧邻上文提供的素材，按本 skill 的六块输出结构生成 `references/weeks/<周次>.md`。没有收到可用素材时才向用户索取。
3. 确保打卡模板要求群昵称或真实姓名，并包含面向新手的 Codex 提交 PR 提示词。
4. 更新 `README.md` 的“当前已收录周次”；如果用户给出的正式标题与日历不同，同步更新日历标题。
5. 检查差异、周次、日期、文件路径和敏感信息。不要覆盖其他周次或无关内容。
6. 在分支提交改动并创建 PR。此固定指令代表维护者已授权正式发布；验证通过后将发布 PR 合并到 `main`，不要停留在草稿状态。
7. 按标题查找 `<周次> 作业评论区`。不存在时创建 Issue，存在时复用，绝不因为编号不同重复创建。
8. 返回已发布周次、主分支文件链接和评论区 Issue 链接。只有文件已合并且 Issue 已存在时才能说“发布完成”。

评论区 Issue 正文应说明：学员只需用 Codex 提交作业 PR；维护者审核后运行“更新打卡统计”；作业默认保存到 `references/submissions/`。

### 更新打卡统计

当用户说“更新打卡统计”时，直接完成以下流程；用户明确指定周次时使用指定周次，否则按当前日期匹配本周：

1. 按标题定位 `<周次> 作业评论区`，不要依赖固定 Issue 编号。若本周任务已发布但 Issue 缺失，则补建一次。
2. 获取仓库全部相关 PR。有效作业优先以 `references/submissions/<周次>-*.md` 为准；兼容早期路径时，PR 标题或正文必须明确写有该周次，且改动必须是单份作业报告。
3. 统计 OPEN 或 MERGED 的有效作业 PR，排除关闭未合并、周任务发布 PR、规则文档 PR和同一账号的重复提交。同一账号有多份时保留最新有效版本。
4. 按“作业正文中的提交人 -> PR 标题中的姓名 -> 文件名中的姓名 -> 已知姓名映射 -> GitHub 账号”的顺序确定展示名，并保留 GitHub 账号用于核对。
5. 同步前检查密码、密钥、邮箱、客户资料、身份证件、本机绝对路径等敏感信息。评论区副本必须脱敏；原 PR 仍有风险时在结果中提醒维护者，不得声称原 PR 已修复。
6. 每份作业使用隐藏标记 `<!-- homework-sync:<周次>:pr-<编号> -->`。标记已存在时更新原评论，不重复新增；不存在时新增一条作业评论。
7. 使用隐藏标记 `<!-- homework-summary:<周次> -->` 维护唯一一条汇总评论，按“姓名（@GitHub账号）- PR #编号”列出名单和总人数。
8. 只同步和统计，不自动合并学员作业 PR。最后返回新增人数、总人数、名单、评论区链接和需要脱敏或人工审核的 PR。

如果 GitHub 不可访问、登录失效或写入失败，明确说明停在哪一步，不要把本地生成或部分同步说成已完成。

## 使用方式

当用户要求发布某一周任务时：

1. 识别周次标识，例如 `2026H2W1`、`2026H2W2`。
2. 优先读取 `references/weeks/<周次>.md`。
3. 如果周次文件不存在，读取 `references/templates/weekly-task-template.md`，根据用户提供的素材生成新周次内容。
4. 输出中文内容，面向微信群/共学群可直接发布。
5. 保持结构稳定，但允许根据当周主题调整任务步骤和验收标准。

当用户要求发布 `2026H2`、`2026H1` 这类半年度任务时：

1. 识别 `H1` 为上半年，`H2` 为下半年。
2. 生成半年度总规划，放在 `references/plans/<半年度>.md`。
3. 半年度规划以周为单位列出主题、目标、产出和难度。
4. 如果已经有具体周任务文件，要在半年度规划里引用它。
5. 半年度规划不替代周任务文件；后续仍应按周补充 `references/weeks/<周次>.md`。

当用户问“本周的共学群任务是啥”“本周打卡任务是什么”“今天该做哪周任务”“本周任务发一下”时：

1. 读取当前日期，按用户所在地或系统上下文的时区判断日期。
2. 优先读取 `references/calendars/<半年度>.md` 做日期到周次的匹配。
3. 核心基准规则：当前这周就是 `2026H2W1`，也就是 `2026-07-06` 至 `2026-07-12`；后续每过一个任务周，周次在这个基准上递增。
4. `2026H2` 采用周一到周日为一个任务周，不按自然年周数重新计算。
5. 匹配到周次后，优先读取主分支中已经存在的 `references/weeks/<周次>.md`。
6. 只有已经合并到主分支的周任务文件才算“维护者已发布”；草稿、未合并 PR、聊天里临时生成的内容都不能当作已发布任务。
7. 如果对应周次文件还不存在，就读取 `references/plans/<半年度>.md` 的周任务总表，输出该周标题、目标和主要产出，并提示“本周详细任务尚未发布，以下来自半年度规划”。
8. 回答时必须明确写出匹配依据，例如“今天是 2026-07-06，落在 2026H2W1（2026-07-06 至 2026-07-12）”。

## 输出结构

发布单周任务时，默认输出这六块：

1. 群公告
2. 本周任务目标
3. 参与者操作步骤
4. 可复制提示词
5. 打卡提交模板
6. 完成标准和复盘问题

如果用户说“只要群公告”“只要打卡模板”“整理成 README”，就只输出对应部分。

发布半年度规划时，默认输出这五块：

1. 半年度群公告
2. 阶段安排
3. 周任务总表
4. 打卡节奏
5. 带班复盘规则

## 周任务内容文件

每周任务文件建议放在：

```text
references/weeks/2026H2W1.md
references/weeks/2026H2W2.md
references/weeks/2026H2W3.md
```

每个周任务文件至少包含：

- 周次标识
- 任务标题
- 群公告
- 任务目标
- 操作步骤
- 提示词
- 提交模板
- 完成标准
- 带班复盘问题

半年度规划文件建议放在：

```text
references/plans/2026H2.md
```

半年度规划至少包含：

- 半年度标识
- H1/H2 含义
- 半年目标
- 阶段安排
- 周任务总表
- 打卡节奏
- 后续维护方式

半年度日历文件建议放在：

```text
references/calendars/2026H2.md
```

日历文件用于回答“本周的共学群任务是啥”“本周任务是什么”这类普通问法，不要只凭感觉猜周次。

## 更新规则

新增一周任务时：

1. 在 `references/weeks/` 新增 `<周次>.md`。
2. 用中文写，直接面向共学群成员。
3. 不要把大段临时素材塞进 `SKILL.md`。
4. 如果任务来自课程、PDF、文章或群讨论，提炼成可执行任务，不要原文搬运。
5. 更新 `README.md` 里的“当前已收录周次”。

学员打卡提交规则：

1. 学员使用 Codex 提交作业 PR 时，必须提供群昵称或真实姓名，二选一即可，不强制公开法定姓名。
2. 姓名应同时出现在作业正文开头、文件名和 PR 标题中，便于评论区同步与打卡统计。
3. 作业文件保存为 `references/submissions/<周次>-<群昵称或真实姓名>.md`。
4. PR 标题使用 `提交 <周次> 作业：<群昵称或真实姓名>`。
5. 一个 PR 只提交一位学员的一份作业，不要修改周任务文件或其他学员的作业。
6. 汇总时优先显示群昵称或真实姓名，并保留 GitHub 账号用于核对。

外部贡献和合并规则：

1. 没有仓库写权限的人不能直接写入主仓库，应 fork 仓库或新建分支后提交 Pull Request。
2. PR 只新增或修改对应周次文件，例如 `references/weeks/2026H2W2.md`，必要时同步更新日历或 README。
3. 维护者合并 PR 后，该周任务才算正式发布。
4. 回答“本周任务是什么”时，只使用已经合并进主分支的周任务文件；不要把未合并 PR 当成正式任务。
5. 如果同一周有多个候选 PR，优先提醒维护者确认，不要自动混合多个版本。

## 风格

保持中文、具体、可执行。少讲概念，多给步骤、模板和完成标准。

好的任务应该让成员看完就知道：

- 今天要打开什么
- 复制什么提示词
- 最后交什么
- 怎么算完成
- 哪些地方不能乱来
