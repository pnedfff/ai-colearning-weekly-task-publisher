# 数据核查流程 — 项目内 skill 方法

## 触发词

当我说"数据核查"时，按以下流程执行。

## 流程

### Phase 1 — 只读扫描（不修改、不生成）

1. 读取用户指定的或项目内的全部数据文件（xlsx、csv、txt、md、docx 等文本可读格式）。
2. 忽略隐藏目录、依赖文件、已生成的产物和备份文件。
3. 列出所有发现的字段不一致、金额单位不一致、计算口径不明、疑似超预算、报告引用错误等问题。
4. 每个发现必须写清楚：文件名、具体位置、原文。
5. 不修复文件，不生成新文件。

### Phase 2 — 提问（每次只问 1 个）

1. 从扫描结果中选出最关键的 5 个口径问题，逐一提问。
2. 每个问题必须带原文和具体位置。
3. 每个问题必须包含：
   - 如果字段有多种写法 → 问统一叫法
   - 如果数字看起来不合理 → 问正常范围或例外条件
   - 如果缺失值不确定 → 问是必填、可空、还是待补充
4. 其他未问到的发现，统一放入"待确认"清单，在后续输出中列出。
5. 不自己判断，不自己修复。

### Phase 3 — 修复（只生成新文件）

1. 根据书面确认的口径生成修复版：
   - 每个修复文件保存为新文件，文件名加 `_修复版` 后缀
   - 不覆盖任何原文件
   - Excel 文件保留公式、格式、sheet 和数据类型
2. 输出 3 份文档到 `outputs/`：
   - `检查日志.md` — 记录每个修复项、关键指标、需人工抽查的例外
   - `对照清单.md` — 原文件与候选修复版的对照关系
   - `自检清单.md` — 可复用的检查标准（含严重程度和例外条件）

## 输出结构规范

### 检查日志.md 必须包含

- 已处理的修复项（修复内容）
- 待人工确认的项
- 关键指标速览表

### 对照清单.md 必须包含

- 原文件 → 候选修复版 的逐文件对照
- 每个对照标注差异内容和是否需要人工确认

### 自检清单.md 必须包含

每个检查项：
1. 检查内容
2. 合格标准
3. 严重程度（必须通过 / 建议通过 / 可选）
4. 需要人工确认的例外情况
