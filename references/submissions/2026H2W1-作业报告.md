## 4. Codex 建议的分类结构

Codex 只读盘点后，建议按用途分为以下 8 个目录：

```
根目录/
├── 01_活动策划与方案/    ← 读书会方案
├── 02_会议记录/          ← 主持稿、会议纪要
├── 03_宣传推广/          ← 公众号推文、小红书文案、海报、直播链接
├── 04_嘉宾资料/          ← 嘉宾介绍 PDF、嘉宾照片
├── 05_现场素材/          ← 活动流程、签到二维码、截图、现场照片
├── 06_数据表格/          ← 报名表、物料清单、用户反馈、预算表
├── 07_过程记录/          ← 录音转写稿、复盘想法
└── 99_待确认/           ← 用途不明或版本冲突的文件
```

## 5. 我确认后执行了哪些整理

1. 确认了分类方案后，Codex 创建了 8 个分类子目录
2. 将 26 个文件按类别移入对应目录（AGENTS.md 保留在根目录）
3. 每个目录按内容主题命名，以数字前缀保证排序
4. 对于版本冲突的文件（会议纪要 x2、方案 x2、嘉宾照片 x2），将多余版本保留在分类目录中，在文档中标注需要确认
5. 对于用途不明的文件，移入 99_待确认 目录等待后续确认

## 6. Codex 生成了哪些说明文档

| 文档 | 用途 |
|------|------|
| README.md | 文件夹结构和使用说明，包含目录对照表和注意事项 |
| 文件索引.md | 所有文件的完整位置映射，文件名、类型、大小、备注 |
| 整理日志.md | 完整的整理过程记录，包括盘点、计划、执行和遗留问题 |
| AGENTS.md | 整理规则文件，限定只读范围和安全边界 |

## 7. 待确认清单里有哪些典型问题

1. 版本冲突：读书会方案 v1 与 v3 修改版需确认最终版
2. 最终版不明：会议纪要两个版本（_final_真的最终 vs 最终版）需确认
3. 照片选择：嘉宾照片原始版与新版的取舍
4. 临时标记：预算表-临时.xlsx 需确认是否有正式版本
5. 无意义命名：IMG_3821.jpg 相机直出文件名，需根据内容重命名
6. 不明用途文件：未命名文档.docx、新建文本文档.txt、老版本不要用.docx 需确认后处理

## 8. 我做的二次调整是什么

在第一轮整理结果上，我让 Codex 做了一次二次调整：

调整内容：在 README.md 中补充版本冲突说明，标注每个版本的最后修改时间和差异方向，便于后续快速决定如何合并。二次调整也遵循了"先列计划、确认后执行"的流程。

## 9. 这次我学到的一个安全使用 Codex 的经验

最有价值的经验是：先写 AGENTS.md 规则文件，再做任何事情。在开始整理之前，先给 Codex 设定明确的边界（只读盘点、禁止删除/覆盖、不确定先问、执行后写日志），让整个整理过程变得可控且可追溯。

另一个关键习惯：先复制副本，再在副本上操作。这样即使出了差错，原始文件仍在，完全没有风险。
