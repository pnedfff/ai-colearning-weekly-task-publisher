## 一、任务理解
本周要求选一个**真实但可公开**的项目资料文件夹（≥3 份、≥2 种格式），用 AI 从中提炼出一套可继续汇报和修改的"汇报包"。重点练四件事：
1. 先**只读盘点**资料；
2. 按**业务问题**整理，而非时间线流水账；
3. 给重要结论**标注来源**；
4. 把**缺失信息**和**材料矛盾**单独列出。

## 三、执行流程（符合本周安全要求）
1. **只读盘点**：先列资料清单，不改动任何源文件（见 `成果/00-只读盘点与分类.md`）。
2. **按业务问题分类**：当前进展 / 关键决策 / 阻塞点 / 下一步 / 跨文件矛盾 / 待确认。
3. **结论标来源**：所有关键数字标注 [01]~[05] 来源文件，可追责可复核。
4. **列缺口与矛盾**：资料中刻意预埋并暴露了 **3 处跨文件矛盾 + 1 处方向分歧**。

## 五、本周心得
- **按业务问题分类**比"按时间线流水账"更适合决策场景，能直接暴露阻塞与矛盾。
- **标来源**让汇报可被追责与复核，也方便发现跨文件矛盾——这是本周最大的收获。
- 造资料时刻意预埋矛盾点，正是为了练习"不回避缺口"：真实汇报里这类矛盾最常见，务必先 reconcile 再对外。
- 安全方面延续上周原则：副本 + 只读先行 + 执行留痕，本次为造资料练习，未触碰任何个人目录。
