## 三、项目说明文档

### 3.1 项目背景

**AI共学群每周任务发布器** 是一个 Codex Skill，服务于一个以 AI 实践为主题的共学社群。社群每周发布一个小型 AI 实操任务（从本地文件整理开始，逐步扩展到资料提炼、文档生成、内容发布、工作流复盘），要求参与者提交真实作业，由带班者审核和统计打卡。

项目的核心痛点：

- 带班者需要每周稳定发布任务，但纯手动操作容易遗漏、格式不统一。
- 学员需要通过 Codex 完成并提交作业，但传统 GitHub 操作门槛偏高。
- 打卡统计依赖人工核对 PR，重复劳动且容易出错。
- 不同周的作业格式不一致，复盘时缺乏可比性。

为此，项目把"发布周任务"和"更新打卡统计"封装成两个 Codex 指令，由 Skill 自动完成流程化操作。

### 3.4 关键设计决策

1. **周次规则**：采用自定义周次（2026H2W1 起始于 2026-07-06），而非自然年周数。好处是与下半年开始对齐，便于两周任务复用同一套周次。
2. **固定仓库**：发布和提交都固定在 `pnedfff/ai-colearning-weekly-task-publisher` 主仓库，而非学员的 fork。学员通过 PR 提交作业，而非直接写入。
3. **两个指令**：带班者只需记忆"发布本周作业"和"更新打卡统计"两个固定指令，降低维护负担。
4. **脱敏要求**：评论区同步时必须脱敏，原 PR 如果有敏感信息只提醒不修改，保持作业原文完整性。

## 四、资料摘要

### 4.1 SKILL.md 核心要点

- 固定两个指令："发布本周作业"和"更新打卡统计"
- 发布流程包括：确定周次、生成内容、更新 README、创建 PR 并合并、创建评论区 Issue
- 统计流程包括：获取 PR、去重、确定展示名、脱敏同步、汇总清单
- 学员提交规则：使用群昵称或真实姓名，PR 标题用"提交 <周次> 作业：<姓名>"
- 输出结构：群公告、任务目标、操作步骤、可复制提示词、提交模板、完成标准、复盘问题
- 风格要求：中文、具体、可执行，少讲概念多给步骤

### 4.2 2026H2 半年规划要点

- 六个月分为六个阶段：基础安全资料整理数据辅助内容生产工作流综合项目
- 每周输出明确：项目说明/会议纪要/学习卡片/周报/对比表/复盘文章等
- 打卡模板包含 7 个问题：任务、素材、AI 步骤、产出、最有用的步骤、需要人判断的步骤、可复用提示词
- 周复盘只抓三样：一个好案例、一个常见问题、一条下周规则

### 4.3 2026H2W1 实例特点

- 教用户安全使用 Codex 处理本地文件：先副本、再规则（AGENTS.md）、只读盘点、计划确认、执行留痕、二次调整
- 包含整套提示词（创建 AGENTS.md、只读盘点、整理方案、二次调整）
- 完成标准清晰：副本操作、规则约束、盘点结果、方案确认、分类结构、待确认清单、二次调整、一条经验
