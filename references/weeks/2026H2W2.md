# 2026H2W2｜快速提案：用 Codex 把散乱资料变成方案

## 群公告

本周任务：选一个真实但可公开描述的项目资料文件夹，让 Codex 从多份 Word、Excel、PDF、截图或会议纪要中提炼信息，生成一套可以继续汇报和修改的方案材料。

这周不追求做出漂亮 PPT，重点是练会四件事：先只读盘点资料；按业务问题整理而不是写流水账；给重要结论标注来源；把缺失信息和材料矛盾单独列出来。

最后应得到一套“汇报包”：汇报摘要、PPT 大纲、演讲提纲和补充材料清单。

## 任务目标

把一个散乱的项目资料文件夹，转成结构清楚、结论有来源、风险有边界、下一步能执行的汇报方案。

本周练习的不是让 AI 自由发挥写漂亮稿，而是让 Codex 跨文件读取材料，围绕“当前进展、关键决策、阻塞点、下一步行动”重组信息，并主动暴露数据缺口和跨文件矛盾。

## 参与者流程

1. 准备一个安全的项目资料副本
   - 可以使用活动资料、销售方案、课程项目、运营复盘、个人项目或公开案例。
   - 原始资料至少包含 3 份文件，最好有两种以上格式。
   - 必须先复制副本，只在副本里操作。
   - 不要使用包含密码、密钥、身份证件、未公开客户资料、真实合同或其他不能公开内容的文件夹。

2. 设置安全和输出规则
   - 让 Codex 先读取已有的 `AGENTS.md`；没有时再创建。
   - 第一轮只读，不移动、不重命名、不修改、不删除原始资料。
   - 所有新结果统一保存到 `成果/` 目录，不只停留在聊天中。
   - 不确定的信息必须标为“待确认”，不能补写或猜测。

3. 做一次只读盘点
   - 盘点文件名、格式、主题、时间范围和可能用途。
   - 说明哪些材料能相互支撑，哪些材料可能冲突，哪些信息缺失。
   - 先给出盘点和计划，等待确认后再生成正式结果。

4. 按业务问题生成汇报摘要
   - 不按文件顺序或会议时间线写流水账。
   - 固定整理为：当前进展、关键决策、阻塞点、下一步行动、跨文件矛盾、待确认信息。
   - 重要结论后注明来源文件；没有证据的内容不得写成确定结论。

5. 把摘要扩展成汇报包
   - 生成 `成果/汇报摘要.md`。
   - 生成 `成果/PPT大纲.md`，写明每页标题、核心信息和建议呈现方式。
   - 生成 `成果/演讲提纲.md`，按“先结论、再证据、再风险、最后行动”组织。
   - 生成 `成果/补充材料清单.md`，列出正式汇报前还需要补齐的证据和确认事项。

6. 做一次人工核对
   - 随机检查至少 3 条重要结论，确认来源标注准确。
   - 检查数字、日期、预算、状态等信息是否在不同文件中存在矛盾。
   - 确认输出没有泄露敏感信息，也没有把推测写成事实。

## 完成标准

只要提交内容能证明下面这些点，就算完成：

- 使用的是安全副本，不是唯一原件。
- 原始资料不少于 3 份，并包含至少两种文件格式；确实没有多格式材料时，说明实际情况即可。
- 第一轮完成只读盘点，没有直接改动原始资料。
- 汇报摘要按业务问题分类，不是时间线流水账。
- 重要结论标注了来源文件，没有把推测写成事实。
- 生成了汇报摘要、PPT 大纲、演讲提纲和补充材料清单。
- 至少人工核对了 3 条重要结论。
- 记录了跨文件矛盾、缺失信息或说明为什么没有发现。
- 作业报告已脱敏，只提交过程和经验，不上传真实项目资料。

不强制要求生成正式 PPT，也不强制截图。能用文字说清楚输入、过程、来源核对、风险和最终产出即可。

## 带班复盘

审核打卡时，重点看三件事：

- 好的提交：结论先行、来源明确、矛盾和缺口没有被掩盖、结果真正保存成文件。
- 需要改进：只罗列文件名、按时间线复述、没有来源标注、把候选方案写成已确定事实。
- 最适合群里讨论的问题：Codex 找出的哪一处材料矛盾，最可能在真实汇报时让你被追问？

周复盘可以统计：

1. 大家最常使用的项目资料类型。
2. 最常见的矛盾类型，例如数字口径、预算、版本、负责人或项目状态不一致。
3. 哪种来源标注方式最容易核对和复用。
